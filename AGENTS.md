# Research Guidelines

This repository is for learning, not feature development.

When studying a project:

- Prefer reading source code over documentation.
- Trace execution paths instead of summarizing files.
- Cite exact files, types, and functions.
- Clearly separate facts from assumptions.
- Update notes and comparisons as new findings are verified.
- Do not refactor or modify cloned projects unless explicitly asked.

The objective is to understand how terminal systems work and identify opportunities for coding-agent-native terminal experiences.
